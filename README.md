# WordPress オリジナルテーマ 架空サイト
&nbsp;  
## 概要
WordPressスキルを証明することを目的として制作しました。
  
本制作では、以下の3機能を中心に実装しています。 
- 通常投稿（Post）
- カスタム投稿タイプ（CPT）
- カテゴリ（Category）  
&nbsp;  
## 制作目的
WordPressにおける以下のスキルを証明することを目的としています。
- テーマ開発の基礎理解
- 投稿データの取得とループ処理
- カスタム投稿タイプの実装
- カテゴリによるデータ分類
- テンプレート階層の理解  
&nbsp;
## 実装内容
### ■ 通常投稿（Post）
- 投稿一覧ページ（archive.php）の作成  
- 個別投稿ページ（single.php）の作成  
- ページネーションの実装  
- 投稿日・タイトル・本文の動的表示
### ■ カスタム投稿タイプ（CPT）
- 「制作実績（case）」のカスタム投稿タイプを追加  
- 専用アーカイブページ（archive-case.php）    
- 専用個別ページ（single-case.php）
### ■ カテゴリ（Category）
- カテゴリ別アーカイブページ（category.php）    
- カテゴリ名の動的表示  
- カテゴリごとの投稿一覧表示  
&nbsp;
## 工夫点
### ■ クライアントが簡単に投稿できる環境の整備
クライアントが専門的な知識を持たなくても更新できる設計を意識しました。  

例えば、投稿詳細ページにおける本文の余白（margin-bottom）は管理画面側で調整させるのではなく、  
テーマ側（HTML/CSS）で統一的に制御しています。  

- クライアントは「文章入力」と「改行操作」のみで投稿が可能  
- 不要なスタイル調整を行う必要がない  
- 更新時のデザイン崩れを防止できる

これにより、運用時の負担軽減と、デザインの一貫性維持を両立させています。
### ■ 共通パーツの分割による保守性向上
ヘッダー・フッター・パーツ要素などの共通部分は分割し、再利用可能な構造で実装しています。   

- 修正時、複数のファイルを触る必要がなく、  
  同デザインを使い回しできる    
- コードが少なくシンプルになり、可読性が向上する  

これにより、将来的な機能追加やデザイン変更を想定し、  
保守性と再利用性を意識した設計としています。
